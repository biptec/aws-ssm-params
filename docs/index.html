AWS Systems Manager Parameter Store

Terminal UI and automation-friendly CLI for SSM parameters.

Inspect, filter, edit, import, and export parameters across one region, many regions, or every enabled region in an AWS account. Keep secret values hidden by default and keep stdout clean for scripts.

Quick start GitHub
$ brew install biptec/tools/aws-ssm-params

What it does

One tool for interactive maintenance and scripted workflows.

Browse and edit

Use the TUI to audit, search, sort, create, edit, and delete parameters.

Import and export

Move data through dotenv, JSON, or YAML with explicit field mapping.

Safe secrets

SecureString values stay hidden until you explicitly request decryption.

Multi-region

Target one region, repeat --region, or scan every enabled region.

Installation

Install with Homebrew or download a release binary.

Homebrew Formula

The Homebrew tap uses a Formula, so the normal CLI install command works without --cask. 

brew install biptec/tools/aws-ssm-params

# Or tap once and install by name:
brew tap biptec/tools
brew install aws-ssm-params

Build from source

Go 1.24 or newer is required. Release CI uses the version configured in the workflow.

git clone https://github.com/biptec/aws-ssm-params.git
cd aws-ssm-params
go build -o aws-ssm-params ./cmd/aws-ssm-params

Quick start

Start with one region and one command.

Open the TUI

Browse parameters in one region.

aws-ssm-params --region eu-north-1 tui

Filter production parameters

A bare path filter is treated as a name filter.

aws-ssm-params \
  --region eu-north-1 \
  --filter '/app/prod/**' \
  tui

Export to dotenv

Write command output to stdout and redirect it to a file.

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/prod/**' \
  export > prod.env

Import a dotenv file

Use --base-path when dotenv keys are relative names.

aws-ssm-params \
  --region eu-north-1 \
  import \
  --format dotenv \
  --base-path /app/prod \
  --summary < .env

Command model

Three commands: tui, export, and import.

There are no separate get or put commands. Use scalar export to read a single value, import to write values, and the TUI for manual editing.

CommandUse it forExample
tui Interactive auditing, editing, creating, sorting, searching, and deletion. aws-ssm-params --region eu-north-1 tui
export Backups, scripts, local dotenv files, JSON/YAML migrations, and scalar values. aws-ssm-params --region eu-north-1 export
import Creating and updating parameters from dotenv, JSON, or YAML stdin. aws-ssm-params --region eu-north-1 import < params.yaml
Flag placement: global flags may be written before or after the command name. Repeated CLI list flags must be repeated. Comma-separated lists are accepted only in the matching environment variable. 
aws-ssm-params --region eu-north-1 export
aws-ssm-params export --region eu-north-1

# CLI lists use repeated flags:
aws-ssm-params --region eu-north-1 --region eu-central-1 export

# Env lists may use commas:
AWS_SSM_PARAMS_REGION=eu-north-1,eu-central-1 aws-ssm-params export

# Native AWS aliases are still supported as lower-priority fallbacks:
AWS_REGION=eu-north-1 AWS_PROFILE=production aws-ssm-params export

Credentials and regions

Use AWS SDK credentials with explicit tool configuration.

The tool uses the AWS SDK for Go. Credentials may come from environment credentials, shared config files, SSO profiles, web identity, or instance metadata. Region selection starts with --region, then AWS_SSM_PARAMS_REGION, then the native AWS_REGION/AWS_DEFAULT_REGION configuration.

One region

aws-ssm-params --region eu-north-1 tui

Several regions

aws-ssm-params \
  --region eu-north-1 \
  --region eu-central-1 \
  export

All enabled regions

aws-ssm-params \
  --profile production \
  --all-regions \
  tui

Filtering

Path-aware extglob filters with AND/OR grouping.

A filter condition is field:pattern. A bare pattern is a name filter. Semicolons inside one filter are AND. Repeated filters are OR.

Filter examples

  • name:/prod/* matches one level only.
  • name:/prod/** matches recursively.
  • name:/prod/@(app|worker)/** matches selected branches.
  • type:SecureString filters by SSM type.
  • value:*token* needs values to be loaded; SecureString needs decryption.
# AND inside one group:
aws-ssm-params \
  --filter 'name:/app/prod/**;type:SecureString' \
  export

# OR with repeated filters:
aws-ssm-params \
  --filter 'name:/app/prod/**;type:SecureString' \
  --filter 'name:/shared/prod/**;type:SecureString' \
  export
PatternMeaning
*Match within one path segment only.
**Match recursively and cross /.
?Match one non-slash character.
[a-z], [!abc]Character class or negated class.
@(a|b)Match one alternative.
?(a|b)Match zero or one alternative.
+(a|b)Match one or more alternatives.
*(a|b)Match zero or more alternatives.
!(a|b)Match anything except the alternatives.

SecureString

Hidden by default, decrypted only when requested.

Existing SecureString values are not decrypted unless you pass --with-decryption. This makes metadata audits safer and prevents accidental secret exposure in stdout or logs.

Audit safely

Export names, metadata, versions, hashes, or lengths without revealing plaintext values.

Edit without revealing

The TUI can replace a hidden SecureString value without displaying the old value.

Decrypt intentionally

Use --with-decryption only for trusted terminals, files, and scripts.

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/prod/api/token' \
  export \
  --with-decryption \
  --output-field value \
  --scalar

Command: tui

Interactive inventory, editing, sorting, and deletion.

FlagEnvironmentPurpose
--with-decryptionAWS_SSM_PARAMS_WITH_DECRYPTIONLoad real SecureString values.
--show-columnAWS_SSM_PARAMS_SHOW_COLUMNShow optional table columns such as type, value, region, sha256.
--sort-byAWS_SSM_PARAMS_SORT_BYInitial sort, repeated in priority order.
--no-confirm-overwrite-fileAWS_SSM_PARAMS_NO_CONFIRM_OVERWRITE_FILESkip local overwrite confirmation.
--no-confirm-write-securestringAWS_SSM_PARAMS_NO_CONFIRM_WRITE_SECURESTRINGSkip confirmation before writing secrets to local files.
--no-confirm-delete-oneAWS_SSM_PARAMS_NO_CONFIRM_DELETE_ONESkip confirmation for deleting one selected parameter.
--no-confirm-delete-allAWS_SSM_PARAMS_NO_CONFIRM_DELETE_ALLSkip confirmation for deleting all visible parameters.
 
# Load an explicit expected inventory from stdin.
cat expected-params.txt | aws-ssm-params --region eu-north-1 tui

# Show metadata columns and start sorted by type, then name.
aws-ssm-params \
  --region eu-north-1 \
  tui \
  --show-column type \
  --show-column tier \
  --show-column sha256 \
  --sort-by type:asc \
  --sort-by name:asc

Command: export

Write stable, scriptable output to stdout.

Export supports dotenv, JSON, YAML, selected fields, field mapping, scalar output, multi-region sorting, and optional SecureString decryption.

FlagEnvironmentPurpose
--output-fieldAWS_SSM_PARAMS_OUTPUT_FIELDInclude selected AWS fields.
--map-fieldAWS_SSM_PARAMS_MAP_FIELDRename AWS fields in file output.
--sort-byAWS_SSM_PARAMS_SORT_BYSort records before writing.
--with-decryptionAWS_SSM_PARAMS_WITH_DECRYPTIONDecrypt SecureString values.
--format—dotenv, json, or yaml. Default is dotenv.
--key-field—Write JSON/YAML as an object/map keyed by a field.
--base-path—Remove a base path from exported parameter names.
--scalar—Write exactly one selected output field as scalar values.

JSON backup

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/prod/**' \
  export \
  --with-decryption \
  --format json > prod-params.json

Scalar value

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/prod/api/token' \
  export \
  --with-decryption \
  --output-field value \
  --scalar

Mapped fields

aws-ssm-params export \
  --format json \
  --map-field name:title \
  --map-field value:text

Audit view

aws-ssm-params \
  --all-regions \
  --filter 'type:SecureString' \
  export \
  --format yaml \
  --output-field region \
  --output-field name \
  --output-field version \
  --sort-by region:asc \
  --sort-by name:asc

Command: import

Create and update parameters from stdin.

Import is upsert by default: missing parameters are created and existing parameters are updated. Use --on-create and --on-update to make it safer or interactive.

FlagPurposeUse when
--formatInput format: dotenv, json, or yaml.Choose parser for stdin.
--map-fieldMap input file keys to AWS fields.Your JSON/YAML schema uses custom names.
--key-fieldUse JSON/YAML object keys as an AWS field.Input is keyed by name or another field.
--base-pathResolve imported names or relativize exported names against a base path.Portable dotenv, JSON, or YAML files.
--on-createskip, error, or ask.Control missing parameter behavior.
--on-updateskip, error, or ask.Control existing parameter behavior.
--continue-on-errorContinue after per-record failures.Bulk imports where one bad row should not stop the run.
--summaryPrint created/updated/skipped/failed counts.You want an import report.
--default-typeDefault type: string, string-list, secure-string.Input does not include type.
--default-tierDefault tier: standard, advanced, intelligent-tiering.Input does not include tier.

Dotenv relative to a base path

aws-ssm-params \
  --region eu-north-1 \
  import \
  --format dotenv \
  --base-path /app/prod \
  --summary < .env

Ask before writes

aws-ssm-params \
  --region eu-north-1 \
  import \
  --format yaml \
  --on-create ask \
  --on-update ask \
  --summary < params.yaml

Create only

aws-ssm-params \
  --region eu-north-1 \
  import \
  --format json \
  --on-update error \
  --summary < new-params.json

Update only

aws-ssm-params \
  --region eu-north-1 \
  import \
  --format yaml \
  --on-create skip \
  --summary < updates.yaml
Import supports one target region at a time. --all-regions and multiple --region values are intentionally not supported for writes.

Formats

Choose dotenv for local files, JSON/YAML for structured migration.

Dotenv

Best for local development and simple secret files. Export includes SSM comments when possible.

# ssm: /app/prod/api/token
# type: SecureString
API_TOKEN="secret"

JSON

Best for scripts, reviewable diffs, and automation. Supports arrays and keyed objects.

{
  "/app/prod/api/token": {
    "type": "SecureString",
    "value": "secret"
  }
}

YAML

Best for human-edited migration files. Supports arrays and keyed maps.

/app/prod/api/token:
  type: SecureString
  value: secret

Logging

Logs go to stderr; command data stays on stdout.

Logging is disabled by default. Enable debug or trace when diagnosing AWS calls. Trace logs include low-level AWS HTTP timing diagnostics. SecureString values are masked in logs. 

aws-ssm-params --log-level debug export > params.json 2>debug.log

aws-ssm-params \
  --all-regions \
  --log-level trace \
  --filter 'name:/app/prod/**' \
  export > params.json 2>trace.log

Scenarios

Practical command patterns.

Metadata-only SecureString audit

aws-ssm-params \
  --all-regions \
  --filter 'type:SecureString' \
  export \
  --format yaml \
  --output-field region \
  --output-field name \
  --output-field version \
  --output-field date

Generate local dev env

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/dev/**' \
  export \
  --with-decryption \
  --format dotenv > .env.local

Migrate selected parameters

aws-ssm-params --region eu-north-1 \
  --filter 'name:/app/prod/**' \
  export --with-decryption --format json > params.json

aws-ssm-params --region eu-central-1 \
  import --format json --summary < params.json

Find cleanup candidates

aws-ssm-params \
  --region eu-north-1 \
  --filter 'name:/app/!(prod|stage)/**' \
  tui \
  --show-column type \
  --show-column date
