aws-ssm-params

A CLI/TUI for AWS SSM Parameter Store discovery, filtering, import, export, and editing.

Commands

aws-ssm-params [global options] <command>

tui
import
export

Global options

--region value       repeat for multiple regions
--all-regions
--profile value
--no-color
--keymap value
--log-level value
--filter value
--filters-file path

Global flags can be written before or after the command name. CLI lists use repeated flags. Comma-separated lists are supported only in environment variables such as AWS_SSM_PARAMS_REGION.

Filters

Filters use field:extglob. A bare filter is a name filter. Conditions inside one filter are joined by AND with ;. Multiple filters are joined by OR.

--filter 'name:/prod/*;region:eu*'
--filter '/app/prod/**'

--filters-file uses the same syntax with one OR group per line. Blank lines and # comments are ignored.

For imports, input data is parsed first, filters are applied to the parsed records, and only then are AWS describe/write requests performed.

JSON fields

--output-field aws_field selects export fields, and --map-field aws_field:file_field maps AWS field names to import/export file field names. JSON and YAML use arrays by default. --key-field name switches JSON to an object or YAML to a map keyed by the selected AWS field. For imports from ordinary dotenv files, --root-path /app/prod prefixes relative keys such as DATABASE_URL.

Import write policy

By default, import creates missing parameters and updates existing parameters. Use --on-create skip|error|ask and --on-update skip|error|ask to change that behavior. --continue-on-error is available for import, and --summary prints final import counts.

Scalar export

export --output-field value --scalar writes one selected field per matching parameter. JSON/YAML scalar export returns arrays, or keyed maps when combined with --key-field.
